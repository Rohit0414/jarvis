Domain Filter (Precision Control)

By default, all data is synced. Use Odoo Domains to filter specific records (e.g., only "Confirmed" orders).

Example Queries
Only Confirmed Sales:
[('state','=','sale')]
Large Orders (>1000) for Company 1:
[('company_id','=',1),('amount_total','>',1000)]
Performance Tips
  • Filters reduce row count, saving bandwidth and Snowflake storage costs.
  • Test filters in Odoo's search bar first; if it works there, it works here.
  • Ensure your date filters are dynamic if you only want recent data.

Fix Problems

Most sync issues are related to configuration or Snowflake permissions. Follow this checklist to resolve common errors.

Permission Check

Your Snowflake user must have rights to CREATE TABLE, INSERT, and MERGE in the specified schema.

Connection Issues

Double-check the Account ID format. It must include the region: xyz123.us-east-1.

Data Types

If you manually altered Snowflake columns, the connector may fail. Delete the table and re-export to reset.

FAQ

Will it delete data?

No. It safely merges updates and adds new records without affecting existing historic data.

Handle big data?

Yes. Data is sent in optimized batches to handle millions of rows without crashing Odoo.

Custom Models?

Absolutely. Works with any standard Odoo model and any custom models created via Studio or code.
