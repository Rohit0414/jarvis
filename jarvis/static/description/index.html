What does this module do?

This module connects Odoo with Snowflake so you can do reporting, analytics, dashboards, and BI in Snowflake — while keeping data synced automatically.

🚀 Export Any Odoo Model

Export Sales Orders, Invoices, Inventory Moves, Accounting Entries, HR, CRM, or any custom tables. Choose exactly which columns you want.

🔀 Incremental Sync

Sync only what changed. The connector tracks write_date + ID and exports only new/updated rows — fast and cost-efficient.

⏱ Cron Automation

Turn on automatic background exports at your chosen interval (e.g. every 15 minutes). Ideal for near real-time reporting.

❄ Snowflake-Native UPSERT (MERGE)

Snowflake tables are auto-created and updated using MERGE so you don’t get duplicates. Updates and inserts happen safely.

Snowflake Connection Setup

Set it once. All exports will reuse the same connection.

Step 1
🛠 Where to configure in Odoo

Go to: Settings → Snowflake Connector Pro

Fill these values:
  • Account (example: xy12345.ap-south-1 or org/account locator)
  • User (Snowflake username)
  • Password (or Key-based auth if enabled)
  • Warehouse (example: COMPUTE_WH)
  • Database (example: ODOO_DB)
  • Schema (example: PUBLIC)
  • Role (optional but recommended)
💡 Recommended: create a dedicated role like ODOO_EXPORT_ROLE and assign only required permissions.
Minimum Snowflake permissions
Your role must be allowed to:
  • Create tables (first export)
  • Alter tables (if new columns appear)
  • Merge (UPSERT updates + inserts)
  • Drop staging tables (cleanup)

Create Export Queries

Each “Export Query” defines: Model + Columns + Filter + Automation settings.

Step 2
What is an Export Query?

Think of it as a saved export configuration. You can create multiple exports: sales_order, sale_order_line, account_move, etc.

Fields explained
  • Export Name → Target table name in Snowflake (recommended: use model name)
  • Odoo Model → Select which table/model to export
  • Columns → Select columns to export (choose only what you need)
  • Domain Filter → Optional filter (export subset of data)
  • Incremental Auto Export → Run automatically via cron
  • Interval Minutes → Sync schedule interval
Recommended naming

Use export names like: sale_order, sale_order_line, account_move. This keeps Snowflake tables clean and predictable.

⚠️ If you rename an Export Query later, it may create a new table in Snowflake.

Advanced Domain Filters

Domain filters help export only the data you need (useful for multi-company, specific states, or selected customers).

Examples
Export only confirmed Sales Orders:
[('state','=','sale')]
Export only Company 1, totals above 1000:
[('company_id','=',1),('amount_total','>',1000)]
Best practices
  • Keep filters simple for better performance
  • Start without filters → confirm export works → then add filters
  • Use filters to reduce Snowflake storage + compute cost
  • Always test with “Run Export” first
💡 Domain filters follow Odoo domain syntax. If unsure, test domain filters in Odoo search first.

Incremental Sync & Automation

Keep Snowflake always updated with minimal workload.

Step 3
How incremental sync works

The connector tracks the last exported cursor using: write_date and ID.

  • If a row is updated, its write_date changes → it exports again
  • If multiple rows have same write_date, ID ensures stable ordering
  • Snowflake MERGE updates existing rows and inserts new rows
Automation tips
  • For large databases: start with 30–60 minutes interval
  • For near real-time reporting: 5–15 minutes interval
  • Enable only for the exports you actually need

Verify Export in Snowflake

After export, you will see a table created/updated in Snowflake.

Step 4
Where the data goes

Tables are created in: DATABASE.SCHEMA.TABLE

  • Database & Schema come from Settings
  • Table name comes from your “Export Name”
  • The connector auto-creates tables on first export
Example validation query:
SELECT COUNT(*) FROM ODOO_DB.PUBLIC.SALE_ORDER_LINE;
What happens behind the scenes
  • Data is first written to a temporary staging table
  • A MERGE happens from staging → target table
  • Staging table is automatically deleted after success/failure (cleanup)
  • If columns change in Odoo, new columns can be added to target table

Performance & Best Practices

Use these tips for stable, fast exports (especially on large databases).

Recommended approach
  • Start with 1 export (example: sale_order)
  • Confirm it appears in Snowflake
  • Then add related exports (example: sale_order_line)
  • Enable automation last (after testing)
Reduce cost & load
  • Select only required columns
  • Use domain filters for old/irrelevant data
  • Keep interval reasonable (15–60 minutes for most cases)
  • Use a dedicated warehouse sized for ETL

Troubleshooting

Most export issues are permissions or configuration values. Use this checklist.

Connection issues
  • Verify Account identifier is correct
  • Confirm Warehouse is running and accessible
  • Confirm Database + Schema exist
  • Confirm Role is assigned to the user
  • Try login in Snowflake UI using same credentials
Permission errors
Ensure your role has:
  • USAGE on warehouse
  • USAGE on database & schema
  • CREATE TABLE / ALTER TABLE / MERGE permissions
  • DROP permission for staging cleanup

FAQ

Does this overwrite data?

No. It uses MERGE-based UPSERT to update existing rows and insert new rows safely.

Is it safe for large datasets?

Yes. Exports run in batches and incremental sync reduces repeated load.

Can I export custom models?

Yes. Any model/table visible in the module can be exported.
