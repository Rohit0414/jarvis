What does this module do?

It connects Odoo to Snowflake. After that, your Odoo data can live inside Snowflake too. Then you can make reports, dashboards, and charts.

🚀 Export Any Model

You can export many things, like:

  • Sales Orders, Invoices, Customers, Products, Inventory, and custom models.
🔀 Export Only New Changes

It does not keep exporting the same old data again and again. New rows and changed rows get exported. This is faster and saves Snowflake cost.

⏱ Auto Export (Schedule)

You can choose a time like every 15 minutes. Then Odoo will export automatically. Good for daily/weekly reporting and real-time dashboards.

❄ Safe Update in Snowflake

It will not create duplicate rows. If row already exists → it updates it. If row is new → it inserts it. This keeps the table clean.

Why do people use Snowflake?

Because Snowflake is great for analytics. It helps companies answer questions like: “How much did we sell this month?” “Which product sells the most?” “Who are our top customers?”

What happens when you click Export?
  • Odoo reads data from your model
  • It sends data to Snowflake
  • Snowflake saves it as a table
  • Next time, it sends only new/changed data
Good first export to test

Start with a small model first, like Customers or Products. After it works, export bigger tables.

Example: res.partner (Customers) or product.product (Products)

Step 1: Setup Connection

Do this one time.

Required Setup
🛠 Where is the Settings page?

Go to: Settings → Snowflake Connector Pro

Fill these fields:
  • · Account: (your Snowflake account name)
  • · User: (Snowflake username)
  • · Password: (Snowflake password)
  • · Warehouse: (example: COMPUTE_WH)
  • · Database: (example: ODOO_DB)
  • · Schema: (example: PUBLIC)
  • · Role: (optional)
How to find your Snowflake Account:

In Snowflake, open a worksheet and run this query:

SELECT CURRENT_ACCOUNT(), CURRENT_REGION();

Combine them as: ACCOUNT.REGION
Example: HN81816.ap-south-1

💡 Pro-Tip: Make sure your Snowflake user can create tables and update data. Ask your admin if you see permission errors.

Step 2: Create an Export

Choose what to send to Snowflake.

Mapping Data

What is an Export?

It is like a saved “export button”. Once you create it, you can run it anytime. You can also create many exports, one for each model you want.

Simple meaning of fields:
  • · Export Name: Table name in Snowflake. Use snake_case (e.g., sale_order).
  • · Odoo Model: Which Odoo model you want to export.
  • · Columns: Which fields you want. Pick only what you need to keep it fast.
  • · Interval: How often it runs in minutes (for Auto Export).
img/02b-columns.png

Domain Filter (Optional)

A domain filter means: “export only some rows”. If you don’t add a filter, it exports everything.

Examples:
Only confirmed Sales Orders:
[('state','=','sale')]
Company 1 & Amount > 1000:
[('company_id','=',1),('amount_total','>',1000)]
Tips:
  • Start without filter to confirm basic sync works.
  • Use Odoo’s list view search to test your filter logic first.
  • Filters help keep Snowflake clean and lower your costs.
Tiny checklist before you run:
  • Did you choose the correct model?
  • Did you select columns?
  • Is Snowflake connection saved?

Step 3: Auto Export (Incremental Sync)

Keep Snowflake updated automatically.

Automation
What is “incremental” sync?

It means: export only what is new or changed. Old rows are not re-sent, saving Snowflake credits and Odoo resources.

Interval Minutes:

Set how often the auto-sync runs. 15 minutes is typical for Sales. Use 60 minutes for large master data.

💡 Manual Override: You can always click “Run Export” manually even if automation is turned off.

What happens inside?

Module sends data to a temporary table → then updates the main table safely → temporary table is deleted automatically. Exporting again is safe; changed Odoo rows will update Snowflake rows.

Fix Problems (Troubleshooting)

Most problems are just wrong settings or missing permissions. Use these checks.

Check these:
  • Account ID: Does it include the region (e.g., id.region)?
  • Warehouse/DB/Schema: Do the names exist exactly as shown in Snowflake?
  • Permissions: Ask your admin to allow CREATE TABLE and MERGE.
⚠ Important: If you changed Snowflake table types before, you may need to delete the table once and re-export.

FAQ

Will it delete my data?

No. It updates old rows and adds new rows safely.

Can it handle big data?

Yes. It sends data in batches and only sends changes.

Works with custom models?

Yes. Both standard and custom models are supported.

Do I need SQL?

No. You just choose model and columns from the UI.

Can I stop auto export?

Yes. Turn off the Auto Export toggle anytime.

Where do I see my tables?

In Snowflake: Database → Schema → Tables.
