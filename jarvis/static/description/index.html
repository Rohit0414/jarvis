Setup Connection

🛠 Find Settings Page

Navigate to: Settings → Snowflake Connector Pro

Connection Fields:
  • · Account: Combine Account ID and Region (e.g., HN81816.ap-south-1)
  • · User: Snowflake login username
  • · Password: Snowflake login password
  • · Warehouse: Your compute engine (e.g., COMPUTE_WH)
  • · Database: Target container (e.g., ODOO_DB)
  • · Schema: target folder (e.g., PUBLIC)
How to find your Account ID

Execute this query in a Snowflake worksheet:

SELECT CURRENT_ACCOUNT(), CURRENT_REGION();
💡 Pro-Tip: Use ID.REGION for the Account field.

Create an Export Profile

What is an Export Profile?

A saved mapping that tells Odoo exactly what to push to Snowflake. You can create multiple profiles for different models.

Profile Settings:
  • · Export Name: Becomes the table name in Snowflake.
  • · Odoo Model: The source data (e.g., sale.order).
  • · Columns: Select fields to include. Fewer columns = Faster sync.
  • · Domain Filter: (Optional) Send only specific rows.
💡
Checklist Before Running:

Ensure the Odoo Model exists and you have selected at least one column for the export.

📷 Export Configuration Form
📷 Detailed Column Selection

Domain Filters (Advanced Control)

Filters allow you to export only relevant data. For example, syncing only "Confirmed" Sales Orders instead of Drafts.

Confirmed Orders:
[('state','=','sale')]
By Company & Amount:
[('company_id','=',1),('amount_total','>',1000)]
Sync Optimization

Start without filters to confirm connectivity. Add filters once successful to minimize warehouse compute time.

3

Automation & Incremental Sync

How Incremental Sync Works

This connector tracks the last sync timestamp. In the next run, it only looks for records updated after that time. This keeps your data warehouse up-to-date with minimal resource impact.

Scheduling Intervals

Define how often the automation runs. We recommend 15 minutes for sales and 60 minutes for inventory/product data.

💡

Note: You can always run a manual export by clicking "Run Export" even if automation is enabled.

Verification in Snowflake

Confirming Your Data Flow

After the export completes, you will find your Odoo records as a structured table in your selected Snowflake database.

-- Run this to check row count:
SELECT COUNT(*) FROM ODOO_DB.PUBLIC.SALE_ORDER_LINE;
Check via Snowflake UI:
  • · Data Preview: Navigate to Data → Databases and select your schema to preview raw rows.
  • · Worksheets: Run SQL queries like SELECT * FROM <TABLE> LIMIT 50; to verify columns.
  • · Staging Logic: Data is safely moved via temporary tables to ensure integrity.

Fixing Common Problems

If the sync fails, check these common configuration blockers.

Validation Checklist:
  • Is the Account ID including the region? (e.g. xyz.us-east-1)
  • Does your Snowflake user have CREATE TABLE permissions?
  • Are your Database and Schema names spelled correctly (Case Sensitive)?
  • Try a manual export for a small model (5-10 rows) first.
