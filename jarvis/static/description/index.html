What does this module do?

It connects Odoo to Snowflake. After that, your Odoo data can live inside Snowflake too. Then you can make reports, dashboards, and charts.

🚀 Export Any Model
  • Sales Orders, Invoices
  • Customers, Products
  • Inventory
  • And also custom models
🔀 Incremental Changes

It does not keep exporting the same old data again and again.

  • New rows exported
  • Changed rows exported
  • Saves Snowflake cost
⏱ Auto Export

Choose a time like every 15 minutes. Odoo will export automatically.

  • No manual clicks
  • Daily/Weekly reports
  • Near real-time data
❄ Safe Updates

It will not create duplicate rows in Snowflake.

  • Updates existing rows
  • Inserts new rows
  • Keeps the table clean
Why do people use Snowflake?

Because Snowflake is great for analytics. It helps companies answer questions like: “How much did we sell this month?” “Which product sells the most?” “Who are our top customers?”

What happens when you click Export?
  • Odoo reads data from your model
  • It sends data to Snowflake
  • Snowflake saves it as a table
  • Next time, it sends only new/changed data
Good first export to test

Start with a small model first, like Customers or Products. After it works, export bigger tables.

Example: res.partner or product.product
Step 1

Setup Snowflake Connection

Do this one time.

🛠 Where is the Settings page?

Go to: Settings → Snowflake Connector Pro

  • Account (your Snowflake account name)
  • User (Snowflake username)
  • Password (Snowflake password)
  • Warehouse (example: COMPUTE_WH)
  • Database (example: ODOO_DB)
  • Schema (example: PUBLIC)
  • Role (optional)
Simple meaning of Snowflake words
  • Warehouse = the “engine” Snowflake uses
  • Database = a big container for tables
  • Schema = a folder inside the database
  • Role = permissions (what the user can do)
💡 Make sure your Snowflake user can create tables and update data. If not, ask your Snowflake admin.

Step 2

Create an Export

Choose what to send to Snowflake.

What is an Export?

It is like a saved “export button”. You can create many exports, one for each model.

Simple meaning of fields
  • Export Name = table name in Snowflake
  • Odoo Model = which model to export
  • Columns = which fields you want
  • Domain Filter = export only some rows
  • Auto Export = run automatically
  • Interval = how often it runs (minutes)
Good export names

Use: sale_order, account_move. Keep names short using letters and underscores.

What columns should you pick?

Pick only columns you really need. This makes export faster (e.g., customer, date, total).

Tiny checklist before you run export
  • Correct model chosen?
  • Columns selected?
  • Filter correct (if used)?
  • Snowflake connection saved?

Domain Filter (Optional)

A domain filter means: “export only some rows”. If you don’t add a filter, it exports everything.

Examples

Only confirmed Sales Orders:

[('state','=','sale')]

Only company 1 and amount > 1000:

[('company_id','=',1),('amount_total','>',1000)]
Tips
  • Start without filter
  • Confirm export works first
  • Then add filter to export less data
  • Use search filters on Odoo list view to test ideas
💡 Filters help keep Snowflake clean and small (lower cost).
Step 3

Auto Export (Incremental Sync)

This helps Snowflake stay updated automatically.

What does “incremental” mean?

It means: export only what is new or changed. Old rows are not re-sent, which is faster and cheaper.

What is “Interval Minutes”?

It is how often auto export runs (e.g., 15 = every 15 minutes).

Auto Export tips
  • Try 15 minutes for normal use
  • Try 30–60 minutes for very big databases
  • Enable only for tables you need
What if I don’t want auto export?

That’s fine. Keep auto export OFF. You can still run export manually anytime.

Step 4

Check Data in Snowflake

After export, you will see a new table in Snowflake.

Where is the data?

It goes to your Database and Schema from Settings. Table name is your Export Name.

SELECT COUNT(*) FROM ODOO_DB.PUBLIC.SALE_ORDER_LINE;
What happens inside?

Module sends data to a temporary table, updates the main table safely, and deletes the temp table.

Easy “Success Check”
  • Open Snowflake Database → Schema
  • Find table with Export Name
  • Run SELECT COUNT(*) to see rows
What if I export again?

Export again is safe. If a row changed in Odoo, Snowflake row will also update.

Fix Problems

Most problems are just wrong settings or missing permissions.

Check these
  • Account, User, and Password correct?
  • Warehouse name correct?
  • Database and Schema exist?
  • User has permissions (create/update/merge)?
If you see “permission error”

Ask your Snowflake admin to allow: create table, update table, and merge data.

⚠️ If you changed the Snowflake table types before, you may need to delete the table once and export again.
Quick Help
  • Try manual export first
  • Export small table first
  • Check Snowflake table exists

FAQ

Will it delete my data?

No. It updates old rows and adds new rows safely.

Can it handle big data?

Yes. It sends data in batches and only sends changes.

Does it work with custom models?

Yes. Custom models are supported.

Do I need SQL?

No. You just choose model and columns.

Can I stop auto export?

Yes. Turn off the Auto Export toggle.

Where do I see my tables?

In Snowflake: Database → Schema → Tables.
