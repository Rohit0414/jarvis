Recommended first step
🛠 Where to configure

Go to: Settings → Snowflake Connector Pro

  • Account (example: XXXX.region)
  • User (Snowflake username)
  • Password / Key Auth
  • Warehouse (ex: COMPUTE_WH)
  • Database (ex: ODOO_DB)
  • Schema (ex: PUBLIC)
  • Role (optional)
💡 Ensure the selected Snowflake role has permissions: CREATE TABLE, ALTER TABLE, MERGE, DROP TABLE.

How export works (with screenshots)

Create an Export Query → choose model & columns → run export → verify in Snowflake

Step 1 — Create Export Query
  1. Open Snowflake Connector Pro → Manage Queries
  2. Click New
  3. Set Export Name (this becomes the target table name)
📷 Screenshot: Create Query
img/02-query-form.png
Step 2 — Choose Model, Columns & Filters
  • Select the Odoo Model you want to export
  • Select the Columns
  • (Optional) Add a Domain Filter to export only matching records
Example domain filter:
[('state','=','done')]
📷 Screenshot: Columns & Filters
img/02-query-form.png
Step 3 — Run Export or Enable Automation
  • Click Run Export to export immediately
  • Enable Incremental Auto Export to schedule background sync
  • Set Sync Interval (Minutes)
✅ Incremental sync exports only new/updated records automatically.
📷 Screenshot: Run Export
img/03-run-export.png
Step 4 — Verify in Snowflake

After export, your table appears in Snowflake under: DATABASE.SCHEMA.TABLE

  • Target table is auto-created if missing
  • Data is upserted using MERGE (no duplicates)
  • Staging tables are created temporarily and then removed

Yes. Batch-based and incremental.

Custom models?

Fully supported.
