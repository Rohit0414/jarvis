What does this module do?

It connects Odoo to Snowflake. That means you can see Odoo data in Snowflake and build reports and dashboards.

🚀 Export Any Model

You can export Sales Orders, Invoices, Stock, Accounting, CRM, and also custom models.

🔀 Export Only New Changes

It sends only new rows and updated rows. This is faster and saves cost.

⏱ Auto Export (Schedule)

You can turn on auto export like every 15 minutes. Odoo will run it in the background.

❄ Safe Update in Snowflake

It will not create duplicate rows. It updates old rows and inserts new rows safely.

Setup Snowflake Connection

Do this one time.

Step 1
🛠 Where is the Settings page?

Go to: Settings → Snowflake Connector Pro

Fill these fields:
  • Account (your Snowflake account name)
  • User (Snowflake username)
  • Password (Snowflake password)
  • Warehouse (example: COMPUTE_WH)
  • Database (example: ODOO_DB)
  • Schema (example: PUBLIC)
  • Role (optional)
💡 Make sure your Snowflake user has permission to create tables and update data.

Create an Export

This is where you choose what to send to Snowflake.

Step 2
What is an Export?

It is a saved “export rule”. You can make many exports like sale_order and sale_order_line.

Simple meaning of fields
  • Export Name = table name in Snowflake
  • Odoo Model = which Odoo model you export
  • Columns = which fields you want
  • Domain Filter = export only some rows (optional)
  • Auto Export = run automatically (optional)
  • Interval = how often it runs (minutes)
Good export names

Use simple names like: sale_order, sale_order_line, account_move.

Domain Filter (Optional)

A domain filter means: “export only some rows”.

Examples
Only confirmed Sales Orders:
[('state','=','sale')]
Only company 1 and amount > 1000:
[('company_id','=',1),('amount_total','>',1000)]
Tips
  • Start without filter
  • Check export works
  • Then add filter if you want
  • Filter helps export less data
💡 If you don’t know domain, you can ask your Odoo developer.

Auto Export (Incremental Sync)

Snowflake stays updated automatically.

Step 3
What does “incremental” mean?

It means: export only what is new or changed.

  • New rows are sent
  • Changed rows are sent
  • Old rows are not re-sent again and again
Auto Export tips
  • Try 15 minutes for normal use
  • Try 30–60 minutes if database is very big
  • Enable auto export only for tables you need

Check Data in Snowflake

After export, you will see a new table in Snowflake.

Step 4
Where is the data?

It goes into your Snowflake Database and Schema.

  • Database and Schema are from Settings
  • Table name is your Export Name
  • First run creates the table automatically
Example query:
SELECT COUNT(*) FROM ODOO_DB.PUBLIC.SALE_ORDER_LINE;
What happens inside?
  • Module sends data to a temporary table
  • Then it updates the main table safely
  • Temporary table is deleted automatically

Fix Problems

If export is not working, try these checks.

Check these
  • Account is correct
  • User and password are correct
  • Warehouse name is correct
  • Database and Schema exist
  • Your Snowflake user has permissions
⚠️ If you changed the Snowflake table types before, you may need to delete the table once and export again.

FAQ

Will it delete my data?

No. It updates old rows and adds new rows safely.

Can it handle big data?

Yes. It sends data in batches and only sends changes.

Does it work with custom models?

Yes. Custom models are supported.
