You can choose what you want to send: which model (like Sales Orders), which columns (like customer name), and then click export. You can also turn on auto export so it keeps sending new changes again and again.

What does this module do?

It connects Odoo to Snowflake. After that, your Odoo data can live inside Snowflake too. Then you can make reports, dashboards, and charts.

🚀 Export Any Model

You can export many things, like:

  • Sales Orders
  • Invoices
  • Customers
  • Products
  • Inventory
  • And also custom models
🔀 Export Only New Changes

It does not keep exporting the same old data again and again.

  • New rows get exported
  • Changed rows get exported
  • This is faster
  • This saves Snowflake cost
⏱ Auto Export (Schedule)

You can choose a time like every 15 minutes. Then Odoo will export automatically.

  • No need to click every time
  • Good for daily/weekly reporting
  • Good for near real-time dashboards
❄ Safe Update in Snowflake

It will not create duplicate rows.

  • If row already exists → it updates it
  • If row is new → it inserts it
  • This keeps the table clean
Why do people use Snowflake?

Because Snowflake is great for analytics. It helps companies answer questions like: “How much did we sell this month?” “Which product sells the most?” “Who are our top customers?”

  • It sends data to Snowflake

Setup Snowflake Connection

Do this one time.

Step 1
🛠 Where is the Settings page?

Go to: Settings → Snowflake Connector Pro

Fill these fields:
  • Account (your Snowflake account name)
  • User (Snowflake username)
  • Password (Snowflake password)
  • Warehouse (example: COMPUTE_WH)
  • Database (example: ODOO_DB)
  • Schema (example: PUBLIC)
  • Role (optional)
How to find your Snowflake Account (Step 1)

In Snowflake, open a worksheet and run this query: 

SELECT CURRENT_ACCOUNT(), CURRENT_REGION(), CURRENT_ORGANIZATION_NAME();
After you run it, Snowflake will show values like:
  • CURRENT_ACCOUNT() = your account id (example: HN81816)
  • CURRENT_REGION() = your region (example: ap-south-1)
  • CURRENT_ORGANIZATION_NAME() = your org name (for reference)
What to enter in Odoo “Account”:
Enter it like this: CURRENT_ACCOUNT().CURRENT_REGION()
Example: HN81816.ap-south-1
✅ This is the easiest way to confirm which Snowflake account you are using.
Where to find your Snowflake Username + Password

In Snowflake, on the left side you will see your username. Use that same username in Odoo Settings.

Your Password is your Snowflake login password.

Create an Export

This is where you choose what to send to Snowflake.

Step 2
What is an Export?

It is like a saved “export button”. Once you create it, you can run it anytime. You can also create many exports, one for each model you want.

Simple meaning of fields
  • Export Name = table name in Snowflake
  • Odoo Model = which Odoo model you export
  • Columns = which fields you want
  • Domain Filter = export only some rows (optional)
  • Auto Export = run automatically (optional)
  • Interval = how often it runs (minutes)
Good export names

Use simple names like: sale_order, sale_order_line, account_move. This makes Snowflake tables easy to find.

Tip: Keep names short. Use letters, numbers, and underscore.
What columns should you pick?

Pick only columns you really need. This makes export faster.

  • For sales: customer, date, total
  • For inventory: product, quantity, location
  • For invoices: invoice number, partner, amount

Domain Filter (Optional)

A domain filter means: “export only some rows”. If you don’t add a filter, it exports everything.

Examples
Only confirmed Sales Orders:
[('state','=','sale')]
Only company 1 and amount > 1000:
[('company_id','=',1),('amount_total','>',1000)]
Tips
  • Start without filter
  • First confirm export works
  • Then add filter to export less data
  • If you are not sure, ask your developer
Easy way to test a filter

In Odoo, use search filters on the list view. If you can filter there, you can usually use the same idea here.

💡 Filters help keep Snowflake clean and small (lower cost).

Auto Export (Incremental Sync)

This helps Snowflake stay updated automatically.

Step 3
What does “incremental” mean?

It means: export only what is new or changed.

  • New rows are sent
  • Changed rows are sent
  • Old rows are not re-sent again and again
What is “Interval Minutes”?

It is how often auto export runs. Example: if you set 15, it runs every 15 minutes.

Auto Export tips
  • Try 15 minutes for normal use
  • Try 30–60 minutes if database is very big
  • Enable auto export only for tables you need

Check Data in Snowflake

After export, you will see a new table in Snowflake.

Step 4
Where is the data?

It goes into your Snowflake Database and Schema.

  • Database and Schema are from Settings
  • Table name is your Export Name
  • First run creates the table automatically
Example query:
SELECT COUNT(*) FROM ODOO_DB.PUBLIC.SALE_ORDER_LINE;
What happens inside?
  • Module sends data to a temporary table
  • Then it updates the main table safely
  • Temporary table is deleted automatically
  • Next time, only changes are sent
Easy “Success Check”
  • Open Snowflake
  • Go to your Database → Schema
  • Find the table with your Export Name
  • Run SELECT COUNT(*) to see rows
How to check exported data in Snowflake (UI)
  • Database Preview: Open Data → Databases, then select your Database + Schema, open the table and preview rows.
  • Worksheets: Open Worksheets, run SQL like SELECT COUNT(*) or SELECT * LIMIT 100.
Tip: On the left side of Snowflake you can also see which user you are logged in with.

What if I export again?

Export again is safe. If a row changed in Odoo, Snowflake row will also update.

Fix Problems

If export is not working, try these checks. Most problems are just wrong settings or missing permissions.

Check these
  • Account is correct
  • User and password are correct
  • Warehouse name is correct
  • Database and Schema exist
  • Your Snowflake user has permissions
If you see “permission error”

Ask your Snowflake admin to allow: create table, update table, and merge data.

⚠️ If you changed the Snowflake table types before, you may need to delete the table once and export again.

FAQ

Will it delete my data?

No. It updates old rows and adds new rows safely.

Can it handle big data?

Yes. It sends data in batches and only sends changes.

Does it work with custom models?

Yes. Custom models are supported.

Do I need SQL?

No. You just choose model and columns.

Can I stop auto export?

Yes. Turn off the Auto Export toggle.

Where do I see my tables?

In Snowflake: Database → Schema → Tables.
